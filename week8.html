Why we use functions

Imagine you have the task or process,which you have defined and you need repeat it more than one time.
Will you rewrite all lines of this task ? For example we have the task to find the maximum element in the array and then delete it .
We will use this task 5 times and reusement all of these lines of code is not efficient.

Functions are used to:

  1. Modularize programs
  2. Increased readability
  3. Easier changes in the code
  4. Team Work

Two main categories of functions:

  • C standard library functions
  • Programmer-defined functons

How do they work ?

Functions are invoked by function calls.
Every function call contains the name of the functon being invoked and the list of values
(can be empty) as an input for the function to get its job done.
In this process we have two parties:

  1. Caller or Callng function ( from where we are calling the function )
  2. Called function (invoked function)
Called function finishes its task and reports back to caller about it.

Properties of function

  • Name (how can the function be called/invoked)
  • Input (information passed in when the function is invoked)
  • Output(information returned by the invoked function when finishes the task)
  • Body(how function maps the input into output)

Function definition

Just like any other variable we have defined so far,functions must also be defined before to be called later.

Function definition has the following syntax:
return_type function_name(type1 par1, type2 par2, .....){
//function body
}

Let's define our square function ,which works only with integers:
---------Syntax---------
return_type function_name(type1 par1, type2 par2, .....){
//function body
}

---------square(x)--------
int square(int x){
return x*x
}

Different types of functons

  1. Input -> return value
  2. Input -> no return value
  3. No input -> return value
  4. No input -> no return value
